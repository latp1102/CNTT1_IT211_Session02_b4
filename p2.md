1.Kiến trúc đề xuất
Công ty nên áp dụng Hybrid Architecture
SOAP → Core Banking (ổn định, bảo mật cao)
REST → Microservices (linh hoạt, hiệu suất cao)
Hai hệ thống giao tiếp thông qua
API Gateway
Adapter Service / Integration Layer
2.Lý do cho từng quyết định
   SOAP cho Legacy
   Hỗ trợ ACID transaction
   Chuẩn WS-* enterprise
   Message-level security
   Phù hợp hệ thống tài chính lâu đời
   REST cho Microservices
   Stateless → scale tốt
   JSON nhẹ → nhanh
   Phát triển nhanh
   Cloud-native friendly
3.Khuyến nghị cho công ty
Không thay thế Legacy ngay lập tức 
Bao bọc Legacy bằng API Layer
Xây dựng dịch vụ mới hoàn toàn bằng REST
Áp dụng API Gateway để thống nhất truy cập
Đây là chiến lược chuyển đổi số an toàn nhất
4.Khái niệm cốt lõi
   SOAP
   Simple Object Access Protocol
   Web Service dựa trên XML
   Hỗ trợ WS-Security
   Hỗ trợ distributed transactions
   Có WSDL mô tả service
   REST
   Representational State Transfer
   Kiến trúc API dựa HTTP
   Stateless communication
   Thường dùng JSON
   Nhẹ, nhanh, dễ mở rộng
